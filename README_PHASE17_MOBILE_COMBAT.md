# WILDS Phase 17：手機戰鬥、設定與同步判讀

本版以 Phase 16 為基礎建立獨立版本，保留原有 Boss、Skin、神武、獎勵、召喚與 v19 存檔資料。

## 玩家可見更新

- 新增主動翻滾。
  - 手機按右下角 `💨`；鍵盤按空白鍵。
  - 優先朝目前移動方向翻滾，靜止時使用角色面向。
  - 消耗 28 體力，持續 0.28 秒，冷卻 0.78 秒。
  - 翻滾期間傷害核心會拒絕所有生物、Boss 與災禍傷害。
  - 按鈕會顯示即時冷卻，成功時播放藍白殘影、音效與可關閉的震動。
- 區域首領的主動技能加入 0.62 秒真實前搖；災禍的彈幕、特殊技能、隕石／震波各自有前搖。
- Boss 從 Phase 1 進入 2／3 時會顯示文字、紫金破裂光環、閃光與鏡頭震動。
- 設定選單可調整：鏡頭震動、手機震動、畫面閃光、傷害數字、音效音量、省電模式。
- 省電模式把 Canvas DPR 限制為 1，並減少重複疊加特效。
- 雲端下載發現本機較新時，不再只顯示確認句，而是並排比較：日期、天數、首領、神武、Skin、日誌獎勵、飛升與碎片。

## 設定與存檔邊界

- 遊戲設定存於 `localStorage` 的 `wilds:settings:v1`。
- 設定不進角色存檔，也不上傳雲端；同一角色可在不同裝置使用不同效能與輔助選項。
- 角色存檔維持 v19，無需遷移，Phase 16 存檔可直接使用。

## 圖片與快取

- `assets/fx_dodge.png`：256×256 透明 PNG，藍白風弧與揚塵翻滾殘影。
- `assets/fx_phase.png`：384×384 透明 PNG，紫金首領階段破裂光環。
- 圖片由內建圖片生成模式產生，提示核心分別為「銀白青藍高速風弧、揚塵、無角色」與「俯視紫金破裂能量環、中空、無角色」，再經 chroma key 去背、alpha 收邊、縮放與壓縮。
- Service Worker 升為 `wilds-v41`。
- 十張 Skin 仍完整包含在發行包，但不再阻塞 PWA 安裝；第一次查看／裝備時由現有 runtime cache 保存。

## 崩潰與契約檢查

`node tools/check-all.js` 依序執行：

1. `node --check` 全 JS 與 Service Worker。
2. 依 `index.html` 真實載入順序執行無頭 harness。
3. DOM、跨模組 API、存檔三鐵律、素材、快取、key 與確定性契約。
4. Phase 16 回歸測試。
5. Phase 17 設定持久化、DPR、音量、翻滾體力、無敵與素材測試。
6. v19 雙備份、校驗與復原測試。

## 部署後實機驗收

- iPhone Safari、Android Chrome 各測試搖桿＋翻滾連續操作至少 15 分鐘。
- 逐項切換閃光、震動、傷害數字與省電模式，確認重新開啟後仍保留。
- 在五個區域首領與兩個世界災禍各觀察一次技能前搖及 Phase 2／3 轉場。
- 以兩台已登入 Firebase 的裝置製造本機較新的情境，確認比較面板內容後分別測試保留本機與使用雲端。
