# WILDS Phase 14：荒野獎勵中心與實體 Skin

本階段建立玩家可見的獎勵收藏入口，並把既有「深淵之神／死亡君王」從能力資料升級成會實際替換角色動畫的傳說 Skin。

## 獎勵中心

- 右上角新增 `🎁`，未讀 Skin 或尚未領取的冒險日誌獎勵會顯示數量提示。
- 整合荒野榮譽階級、冒險日誌待領獎勵、傳說 Skin 收藏與神格碎片。
- 荒野榮譽由已領取任務、擊敗首領、Skin 收藏和飛升碎片回推，不增加可能遺失或失衡的新貨幣。
- Skin 卡片會顯示解鎖條件、專屬能力、目前角色預覽，以及裝備／卸下操作。
- 舊存檔若已擊敗災禍但 Skin 狀態缺漏，會依災禍紀錄安全補發。

## 角色專屬 Skin 動畫

每張素材皆為 `640×384` RGBA PNG，維持 `5×3`、每格 `128×128` 的角色動畫契約：

- `assets/player_abyss.png`：星光公主・深淵之神
- `assets/player_death.png`：星光公主・死亡君王
- `assets/player2_abyss.png`：黃豆・深淵之神
- `assets/player2_death.png`：黃豆・死亡君王

切換角色時，已裝備 Skin 會自動切換成該角色專屬動畫；卸下後恢復原始造型。手持工具、神武與翅膀仍由原本前後景裝備層繪製。

## 解鎖與能力

- 擊敗萬眼巨鯤：解鎖「深淵之神」，週期性釋放深淵脈衝。
- 擊敗骸骨泰坦：解鎖「死亡君王」，週期性召喚四向靈魂骨刺。
- 解鎖時自動裝備，之後可從獎勵中心自由切換或卸下。

## 存檔與離線

- 存檔版本升為 `v17`，新增 `rewards.seen`，遵守 collect／apply／migrate 三鐵律。
- Skin 持有與裝備狀態沿用原有 `skins` 存檔，不破壞既有玩家收藏。
- Service Worker 升為 `wilds-v38`，四張 Skin 與 `js/rewards.js` 均已加入離線快取。

## 素材生成規格

素材使用內建圖片生成流程，以 `player.png`／`player2.png` 分別作為編輯參考。提示要求保留角色身份、比例、方向、動作格、腳底錨點和裝備可讀區域，只改造服裝與主題；生成後以色鍵去背、縮放至專案尺寸並檢查 alpha。

## 驗證

```bash
node tools/check-all.js
```

另有 smoke test 驗證災禍進度補發、兩角色素材配對、Skin 裝備、榮譽計算、已讀狀態與 v17 遷移。
