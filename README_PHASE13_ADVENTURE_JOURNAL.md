# WILDS Phase 13：冒險日誌與主線目標

本階段在 Phase 12 的穩定化基礎上加入可見、可追蹤、可保存的冒險進程。既有的素材修正、五位區域首領、世界災禍、導航、快速移動與角色可視裝備均保留不動。

## 玩家可見更新

- 右上角新增 `📜` 冒險日誌按鈕。
- 畫面上方會持續顯示目前主線目標與即時進度。
- 共 5 章、12 個目標，涵蓋採集、建造、夥伴、五位區域首領、神武、祭壇、兩位世界災禍與飛升輪迴。
- 已完成目標的獎勵必須手動領取，避免舊玩家一讀檔就被大量獎勵塞滿背包。
- 背包放不下時會嘗試存入共用倉庫；兩邊空間都不足便不會領取，也不會遺失獎勵。
- 首領與祭壇目標可按「導航目標」，直接接到角色身上的指引箭頭。
- 舊存檔會依目前實際狀態補登已完成目標，不需要重玩已做過的內容。

## 資料與離線更新

- 存檔版本升為 `v16`，新增 `journal.completed` 與 `journal.claimed`。
- 遵守存檔三鐵律：`collect()`、`apply()`、`migrate()` 三處皆已接上。
- v15 與更早存檔會自動遷移；不同世界種子不沿用日誌狀態。
- Service Worker 快取版本升為 `wilds-v37`，`js/journal.js` 已列入核心快取。

## 崩潰檢查

依 `WILDS_CRASH_CHECK_PLAN.md` 執行：

```bash
node tools/check-all.js
```

檢查涵蓋語法層、載入層與結構契約層；另以根目錄 smoke test 驗證任務補登、獎勵領取、匯出還原與祭壇導航。
