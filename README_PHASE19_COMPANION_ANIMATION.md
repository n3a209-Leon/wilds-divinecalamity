# WILDS Phase 19：夥伴動畫與透明邊緣重製

本版從 Phase 18 建立獨立副本，專門修正夥伴移動僵硬、攻擊沒有動作，以及舊 PNG 外圍白邊看似去背不完整的問題。角色存檔維持 v19，Phase 18 進度可直接讀取。

## 新夥伴素材

- `assets/mate_knight_sheet.png`
- `assets/mate_archer_sheet.png`
- `assets/mate_cat_sheet.png`
- `assets/mate_sprite_sheet.png`

每張都是 `576×192` RGBA PNG，由三個 `192×192` 方格組成：待機、移動、攻擊。四張素材均具備透明角落、幀間透明間隔，且已移除綠幕、白色貼紙框、棋盤殘影與灰色霧邊。

圖片以原本兩張夥伴圖作為角色身份與風格參考，使用內建圖片編修模式重製為平面綠幕三幀動畫，再以 chroma-key soft matte、despill、固定比例縮放與腳底基準線完成遊戲格式。原本八張圖片仍保留在專案中作歷史素材，但遊戲只使用新的 `_sheet` 圖，因此不會在載入期間短暫顯示舊白邊。

## 動作系統

- 隊形會依玩家面向旋轉，四名夥伴不再在玩家轉彎時突然互換左右位置。
- 跟隨改為 arrive steering：先加速、接近時減速，避免滑動、瞬停和硬切方向。
- 面向使用平滑插值，並加入待機呼吸、移動上下步伐、輕微前傾與壓縮伸展。
- 超過安全距離才使用防丟失傳送；一般距離不會瞬移。
- 騎士與小貓使用近戰距離，弓手與精靈使用遠程距離。
- 攻擊會先鎖定最近生物、區域首領或世界災禍，再切換第三幀並記錄命中位置。
- 畫面回饋分為騎士劍弧、弓手箭軌、小貓三道爪痕與精靈紫色魔法線。

## 載入與相容性

- 四張動畫表登錄於 `art.js`，第一次看見夥伴時才載入。
- Service Worker 更新為 `wilds-v43`，夥伴動畫表使用既有 runtime cache，不阻塞首次安裝。
- 新增欄位全是即時動畫狀態，不寫入角色存檔，因此不需要提高存檔版本。

## 驗收

執行：

```bash
node tools/check-all.js
```

Phase 19 專項會逐像素解碼四張 PNG，確認 RGBA、尺寸、透明角落、幀間透明間隔與無綠幕殘留；並實際驗證夥伴加速度移動、非瞬移、自然動畫狀態、目標搜尋和攻擊幀切換。
