# WILDS Phase 18：戰鬥深度、無障礙與安全更新

本版以 Phase 17 的獨立副本開發，保留五區域首領、世界災禍、十套角色專用 Skin、可視化裝備與 v19 存檔相容性。

## 玩家可見更新

- 完美閃避：翻滾前段成功避開傷害時，生命不減、返還 12 體力，並在 2.5 秒內讓下一擊增加 18 傷害。
- 完美閃避會顯示金色攻擊按鈕、短暫慢動作、專用音效、殘影與觸覺回饋。
- 設定新增「減少動態效果」：即時停用慢動作、命中停頓、震屏與全畫面閃光。
- 設定新增「高對比警示」：Boss 前搖、地面池、火柱與隕石範圍會補上黑白外框。
- 全螢幕背包、商店、製作、旅行、裝備、日誌、獎勵、設定、診斷、雲端比較與更新面板開啟時，世界暫停且輸入鎖定。
- 診斷面板可匯出 `WILDS-save-YYYY-MM-DD.json`，匯入前會限制大小、解析 JSON、確認覆蓋並使用既有 v19 校驗／遷移流程。
- 靜音按鈕會與設定中的音效音量同步，音量為零時正確顯示靜音狀態。

## 載入與 PWA

- `art.js` 開局只預載常見環境、生物、建物與必要回饋圖；Boss、災禍、夥伴、神武等素材第一次需要時才載入。
- Service Worker 升為 `wilds-v42`；首次安裝只預快取啟動素材，其餘由既有 runtime cache 保存。
- 新 Service Worker 安裝完成後不會立即 `skipWaiting`。畫面會先提示玩家，按下「存檔並立即更新」成功保存後才切換並重整。
- 角色存檔維持 v19；Phase 17 進度可直接讀取，不需要新增遷移。

## 崩潰與契約檢查

執行：

```bash
node tools/check-all.js
```

依序驗證語法、真實腳本載入順序、DOM/API/存檔/素材/快取契約、Phase 16～18 行為回歸，以及 IndexedDB 主存檔與兩份輪替備份的非同步復原。

Phase 18 專項另驗證：完美閃避與強化下一擊、無障礙設定持久化、低動態立即清除停頓、輸入鎖定、素材按需載入、PWA 安全切換、備份介面與匯入校驗路徑。

## 部署後實機驗收

- iPhone Safari／Android Chrome 各測試一般翻滾、早期完美閃避、強化下一擊與體力返還。
- 在熔岩魔神與兩個世界災禍戰鬥中開關高對比與減少動態效果。
- 戰鬥中開啟背包與設定，確認角色、敵人、投射物、飽食與世界時間完全暫停。
- 安裝舊版 PWA 後部署新版，確認更新提示不會強制中斷，按確認後先存檔再重整。
- 匯出備份、改動進度、再匯入，確認校驗成功且角色位置、首領、神武、Skin 與日誌一致。
