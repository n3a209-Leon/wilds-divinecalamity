# WILDS Phase 25：野獸生態與顯示修復

## 玩家能直接感受的更新

- 修正野豬、熊、烏鴉雖有素材與生成資料，卻因缺少顯示高度而可能完全畫不出來的問題。
- 鹿、兔、狼、野豬、熊、烏鴉共六種原有野獸，現在都有待機／移動圖、有效高度、透明素材與向量備援。
- 新增狐狸、山羊、巨獾，共三種新野獸與六張正式透明素材。
- 狐狸白天保持距離，夜裡或受擊後會繞行撲擊，並且怕火。
- 山羊主要出沒岩地，靠近後先出現警示，再鎖定方向衝撞；衝刺中不會作弊轉向。
- 巨獾平時中立，受攻擊後會比野豬追擊更久。
- 寬幅奔跑姿勢使用較大的畫面剔除邊界，野獸不會在螢幕邊緣突然跳出或消失。
- 快速移動會依新野獸是否真的進入攻擊狀態判斷，不會被遠處中立生物無故阻擋。
- 老皮與四名招募夥伴只會主動鎖定威脅、秘境守衛或已被玩家攻擊的野獸，不再沿路屠殺和平動物。
- 擊殺訊息會依種類顯示實際掉落數量；烏鴉與陰影不再錯誤顯示生肉、毛皮。

## 顯示安全

- `ART_MOB_H` 已與十種生物索引完整對齊；陰影仍使用原有向量繪製。
- `drawArt()` 增加非數值／零高度保護，未來新增生物即使漏填高度，也會以安全預設值顯示。
- 十八張實體野獸 PNG 全部具有透明通道，並同時登錄於 `art.js` 與 Service Worker。
- 素材載入失敗時仍會顯示依種類著色的簡易向量野獸，不會隱形或造成白畫面。

## 新素材規格

使用內建影像生成，參考既有鹿、狼、熊的手繪素材；每種動物在同一張來源圖中並排生成待機與動作姿勢，再以色鍵移除背景、逐隻裁切及壓縮為遊戲 PNG。

共同提示規格：右向側視完整全身、Q 版手繪、乾淨平塗、深色描邊、左上光源、單一動物兩姿勢、無文字／框線／陰影／浮水印、角色不得被裁切。三組主題分別為紅狐狸「警戒／繞行奔跑」、山羊「站立／低頭衝撞」、巨獾「警戒／受傷後追擊」。

## 存檔與快取

- 一般野獸仍是離開後重新生成的暫態物件，不寫入存檔，因此存檔格式維持 v22，舊進度不需要遷移。
- Service Worker 快取版本升為 `wilds-v55-wild-ecology`。

## 自動驗證

`node tools/check-phase25.js` 會逐種檢查 TYPE 索引、顯示高度、十八張透明 PNG、素材登錄、離線快取、物件池生成，以及狐狸／山羊／巨獾的差異化行為。

完整檢查：

```bash
node tools/check-all.js
```
