# WILDS Phase 16：戰鬥手感與發布強化

本版以 Phase 15 為基礎，只修改新功能涉及的模組，保留既有 Boss、神武、Skin、快速移動與素材整合。

## 玩家可見更新

- 世界災禍祭壇改為真正的 10 秒召喚儀式。
  - 儀式期間再次互動可取消。
  - 離開祭壇範圍或死亡會自動中止。
  - 祭壇顯示災禍名稱、剩餘秒數、進度環與紅色預警法陣。
- 戰鬥加入短暫命中停頓、輕量鏡頭震動與實體命中火花。
- 熔岩柱、災禍隕石與骨刺增加透明預警法陣，危險範圍更容易判讀。
- 快速移動抵達時顯示青藍紫傳送法陣。
- 荒野榮譽不再只有階級文字，會依序解鎖：
  - 60：旅人金色足跡。
  - 160：獵王強化命中特效。
  - 320：紫晶快速移動法陣強化。
  - 600：飛升冠冕。
- 不死鳥由「整份存檔只能復活一次」調整為「每次飛升輪迴可涅槃一次」。

## 存檔安全

- 存檔版本升為 `v19`。
- 每次寫入保留最近兩份 IndexedDB 輪替備份。
- 新存檔附帶校驗碼；主存檔異常時依序嘗試備份 1、備份 2。
- 手動存檔、自動存檔與切出頁面的寫入會依序排隊，避免同時覆寫備份。
- 雲端下載亦先通過校驗與遷移，再輪替本機備份。
- `v18` 的不死鳥旗標會轉換為當時的飛升輪迴編號。

## 新圖片

- `assets/fx_warning.png`：384×384，Boss／災禍攻擊預警。
- `assets/fx_hit.png`：256×256，近戰、弓箭與神武命中火花。
- `assets/fx_travel.png`：384×384，快速移動抵達法陣。

三張圖片使用內建圖片生成流程產生，經純色背景去除、alpha 邊緣收縮、縮圖與 PNG 壓縮後整合。程式仍保留向量效果，圖片尚未載入時不會造成白畫面。

## 離線與檢查

- Service Worker 升為 `wilds-v40`，三張新圖片都加入容錯快取。
- `node tools/check-all.js` 依序執行：
  1. 全 JS 語法檢查。
  2. 依 `index.html` 真實順序執行無頭載入。
  3. DOM、API、存檔、素材、快取與確定性契約檢查。
  4. Phase 16 召喚、榮譽、不死鳥、命中停頓、遷移與圖片煙霧測試。

## 部署後建議實機驗收

- iPhone Safari 與 Android Chrome 各遊玩至少 15 分鐘。
- 測試祭壇開始／手動取消／離開範圍取消／完成召喚。
- 測試鎖屏、切背景、重新開啟與離線啟動。
- 人工破壞測試環境中的主存檔後，確認診斷面板顯示由備份復原。
